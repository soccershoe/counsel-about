The seam

The file is the API.

Docket and Inner Counsel share exactly one thing: a folder of Markdown. No shared code, no IPC, no database. Either app works completely without the other.

Docket

writes the notes

Everything you capture — and everything the connectors distill — lands as plain Markdown in a folder you chose.

~/Vault
.md · .markdown · .txt
the codeless seam
Inner Counsel

reads the folder

It indexes the same folder, deliberates, and writes briefings back into a read-only Counsel/ section.

←  briefings

Data flows one way at a time and never loops. The only cross-app write is a single config pointer that keeps the two pointed at each other across a move — never note data.

Local-first, by architecture

It works for you — not the people who built it.

Nothing leaves your machine

No telemetry, no analytics, no accounts. No webfonts or UI network requests — fonts are bundled or from the system. The only outbound traffic is the connector URL you pasted in.

The file is the source of truth

Plain Markdown on your disk is the canonical store — not a database, not the cloud. Docket is a lens over your files, not a silo that traps them. Stop using it and lose nothing.

No credentials, ever

There is no credential store at all. A personal tool that needs an API key doesn't fit the free, keyless tier — the boundary decides, not convenience.
