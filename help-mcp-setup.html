MCP-client setup

Point any client at your vault.

Docket ships a read-only MCP server — docket-mcp — that hands any Model Context Protocol client typed access to your notes, tasks, and calendar, straight from the same folder. It's read-only and it never leaves your machine.

§What it
is

A read-only accessor over the folder.

The helper ships inside the app at Docket.app/Contents/MacOS/docket-mcp. Your client launches it on demand; it speaks stdio JSON-RPC and reads the vault path Docket records when you open a vault — so there's usually nothing to configure but the path to the helper itself.

It is strictly read-only: there are no write tools, it never follows paths outside the vault, and it skips the Inner Counsel Counsel/ briefings. The folder stays the source of truth — this is just a typed window onto it.

Connect a client

One line, or one config block.

In Docket, Settings → Inner Counsel → Model Context Protocol has a “Copy claude mcp add command” button that fills in the exact path for you. Or set it by hand:

Claude Code — Terminal
claude mcp add docket -- \ /Applications/Docket.app/Contents/MacOS/docket-mcp
Claude Desktop — claude_desktop_config.json
{ "mcpServers": { "docket": { "command": "/Applications/Docket.app/Contents/MacOS/docket-mcp" } } }
Cursor — ~/.cursor/mcp.json
{ "mcpServers": { "docket": { "command": "/Applications/Docket.app/Contents/MacOS/docket-mcp" } } }

Add --vault /abs/path after the command to point at a folder other than the one Docket has open (useful for testing).

I.The tools
it exposes

Six read-only tools.

get_daily_note · list_notes · read_note · search_notes · list_tasks · get_status.

get_daily_note is the structured one — it returns your prose, the connector sections (calendar, mail, …) split out by type, and the day's tasks (each with how many times it's been rescheduled). The rest are exactly what they sound like. All honor the same contract: .md/.markdown/.txt only, a 1 MB-per-file guard, and the Counsel/ folder kept private.

II.Verify

Confirm it's talking.

Restart the client, then ask it to run get_status — you should get back your vault path, note count, and the last daily note. If the tools list but a call errors, open a vault in Docket once so it can record the path.
