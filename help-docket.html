Docket how-to

Set up your vault, turn on the connectors you want, and capture your day — the input lane the rest of the suite reads.

I.Install &
the vault

Pick a folder to be your vault.

Download Docket from the download page, drag it to Applications, and open it. On first run it asks for one thing: a folder to keep your notes in.

That folder is the vault — plain Markdown files you own, readable by anything. It's the one folder Inner Counsel and Slate read, so a plain spot in your home directory (synced or not, your call) is ideal. If you already use Inner Counsel, Docket offers to use its folder on first run, so the two never split.

II.Write
your day

Daily notes, tasks, PARA.

Each day gets a dated note. Write prose, and add tasks with * [ ] — check them off with [x]. Reschedule one forward and Docket quietly stamps how many times it's slipped.

Longer-lived material lives in PARA folders — Projects, Areas, Resources, Archives — beside the daily notes. Everything is just Markdown on disk; there's no database to back up.

III.Pull in
context

Seven connectors, opt-in.

In Settings, turn on the sources you want: Calendar, Reminders, Mail, Apple Notes, Safari Reading List, ICS feeds, and RSS/Atom feeds. They fold scattered context into the day's note in clearly-marked blocks Docket owns and refreshes.

Everything runs locally, with no stored credentials — the connectors read what's already on your Mac. Each is off until you switch it on.

IV.Capture &
read back

Quick-capture, and the Counsel sidebar.

Drop a thought in without breaking flow, and it lands in today's note. When Inner Counsel writes a briefing, Docket shows it read-only in a Counsel sidebar — never copied into your notes, so it can't loop back into the deliberation.

Want another AI tool to read the vault too? Connect an MCP client →

V.If something's
off

When it doesn't do what you expect.

A connector section is empty. Each connector reads what's already on your Mac, so the first time you turn one on macOS asks permission — for Calendar, Reminders, Mail, Notes. If a section stays empty, check that the connector is switched on in Settings and that you granted access.

Inner Counsel can't see my vault. Open Settings → Inner Counsel and click Connect. That adds this vault to Inner Counsel as a source — alongside any notes folder it already reads — so it never disturbs notes you keep elsewhere. If you already use Inner Counsel, Docket offers to use its folder on first run.

Where does my data live, and how do I back it up? It's just the vault — plain Markdown files in the folder you chose. Back it up by copying that folder; there's no database and nothing to export. App settings live in ~/Library/Application Support/Docket/.

I moved my vault in Finder and things broke. Move it from inside Docket (Settings → Vault → move) so the connection follows. If you already moved it in Finder, just point Docket at the new location and reconnect.
