Inner Counsel

A private panel of advisors that reads the context of your work, deliberates across two rounds, disagrees, and reports its findings to one person — you. Entirely on your own machine.

Free · Local-first · macOS
§A panel,
not a prompt

Ten lenses, one quiet conversation.

A single prompt can summarise. To actually advise, you need multiple perspectives — and disagreement. Inner Counsel convenes ten Counselors, each reading the same context through a different lens: evidence, risk, strategy, stakeholders, growth, wins, precedent, load, lateral ideas, the long horizon.

They write independently first, then read each other and react. The Auditor pushes back on the Strategist; the Diplomat flags who'd be alienated; the Troubleshooter remembers you tried this in October. The disagreements are the point. Meet the roster →

I.The
deliberation

Independent first. Reactive second. Synthesised last.

Round one: each Counselor analyses the same context without seeing the others. Round two: each reads every other's take and reacts from its own lens. Then the Orchestrator weights the considered second round and writes a one-paragraph synthesis you can read in under a minute.

The full deliberation is preserved for the days you want to dig — a Key Tensions section keeps the sharpest disagreements visible rather than averaging them away. How a session runs →

II.Five
clocks

Daily to yearly — arcs, not incidents.

Daily reads raw context and asks what happened and what's next. Everything longer — weekly, monthly, quarterly, yearly — reads the previous summaries, not the raw days. That's how the system sees trajectory instead of noise.

The weekly asks what kept getting deferred; the quarterly checks career trajectory and gathers wins for self-review; the yearly asks who you were at the start versus now. The system gets more valuable the longer it runs — not because the model gets smarter, but because the context gets richer.

III.Yours, by
architecture

Local-first. Private. Free forever.

Inner Counsel reads the same shared vault Docket writes, runs a local model by default, and writes its briefings back into a read-only Counsel/ section — which it excludes from its own index, so nothing loops back on itself.

No accounts, no telemetry, no server. The only network calls are to a model on your own machine. Nothing leaves unless you explicitly choose to publish it. How the suite fits together →   The privacy boundary →
